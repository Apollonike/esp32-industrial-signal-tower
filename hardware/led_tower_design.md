# LED Signal Tower – Hardware Design

## Overview

This document describes the hardware design of the network-capable LED signal tower controller. Supports external 12 V supply or PoE-based deployment.

The system is built around an **ESP32-C3** and is designed to integrate a simple LED signal tower into existing automation environments such as:

* Home Assistant
* IoT platforms
* monitoring systems
* industrial control environments

The primary design goal is **reliable system integration with minimal hardware complexity**.

Universal support for arbitrary industrial signal towers is **not the main objective** of the current prototype.

---

# System Goals

The prototype hardware aims to achieve the following:

* network-capable signal tower controller
* ESP32-C3 controller
* three independent LED outputs
* simple hardware architecture
* minimal component count
* robust operation
* easy integration into existing systems

---

# Hardware Architecture

The system is based on a **12 V system rail**.

```
12 V system rail
 ├── LED tower
 └── Step-Down → ESP32-C3 supply
```

The LED segments are switched directly on the **12 V rail** using transistor outputs.

---

# Design Evolution

During early development stages the electrical characteristics of the
LED tower were not fully known.

Prototype measurements later showed that the selected LED tower:

- operates directly from **12 V**
- has very low current consumption
- can be wired using **common +12 V**

These findings enabled a very simple and efficient hardware design.

The final design uses a **12 V system rail** for the signal tower and a
**step-down converter** for the ESP32 controller supply.

---

# Final Architecture

The final design switches the LED segments directly on the **12 V rail**.

Advantages:

* simpler circuit design
* fewer components
* higher efficiency
* easier debugging
* better reliability

---

# LED Tower Characteristics

The prototype uses a **generic no-name LED signal tower**.

Important observation:

The LED modules operate correctly **independent of polarity**.

This means the tower can be wired either with:

* common ground
* or common +12 V

For the final design the following configuration was chosen:

```
Common +12 V
Low-side switching
```

This enables a very simple transistor driver stage.

---

# Output Stage Concept

Each LED segment is controlled using a **low-side NPN transistor switch**.

Simplified circuit:

```
             +12 V
               │
               │
          LED Segment
               │
               │
             Collector
                │
             NPN Transistor
                │
             Emitter
                │
               GND

ESP GPIO ── 2.2 kΩ ── Base
                     │
                   100 kΩ
                     │
                    GND
```

---

# Output Channels

The system provides three independent outputs:

| Channel    | Function       |
| ---------- | -------------- |
| LED_RED    | red status     |
| LED_YELLOW | warning status |
| LED_GREEN  | ready / OK     |

Each channel is controlled by an ESP32-C3 GPIO pin.

---

# Power Supply

The system expects a **12 V supply rail**, typically generated by a PoE splitter or external DC supply.

```
12 V supply
 │
 ├── LED tower
 │
 └── Step-Down → ESP32
```

Recommended additions:

* 100 µF capacitor on 12 V rail
* 100 nF decoupling capacitor near ESP32

---

# Design Philosophy

The hardware design prioritizes:

* simplicity
* robustness
* minimal component count
* easy maintainability

Given the extremely small current consumption of the LED tower, a simple transistor driver stage provides **more than sufficient electrical margin**.

# Home Assistant Integration

This directory contains example configurations and automations for
integrating the ESP32 Industrial Signal Tower into Home Assistant.

The goal is to provide **ready-to-use building blocks** that allow
quick setup without requiring manual configuration from scratch.

---

# Overview

The signal tower is designed to be controlled by Home Assistant.

Typical responsibilities of Home Assistant include:

- evaluating monitoring data
- deciding which signal state should be displayed
- supervising device connectivity via heartbeat
- triggering actions and automations

The ESP32 device itself focuses on:

- executing signal patterns
- monitoring heartbeat timeouts
- providing a simple control interface

---

# Heartbeat Supervision

## Concept

The system uses an **active heartbeat mechanism** to ensure that the
signal tower reflects valid system information.

- Home Assistant sends periodic heartbeat signals
- the ESP32 monitors the heartbeat
- missing heartbeat triggers a fallback signal

This prevents the tower from displaying outdated or misleading states.

---

## Heartbeat Automation

The heartbeat is generated using a Home Assistant automation.

📄 See full configuration:

- [Heartbeat Automation](heartbeat_automation.yaml)

---

## Automation Behavior

The automation performs the following logic:

- runs every second
- checks if heartbeat supervision is enabled
- triggers the `Heartbeat Ping` button on the device

This ensures continuous validation of:

- Home Assistant availability
- network connectivity
- ESPHome API communication

---

## Example Logic


Every 1 second:
IF Heartbeat Supervision = ON
THEN send Heartbeat Ping

# Integration Workflow

A typical setup in Home Assistant consists of:

1. **Device Integration**
   - ESPHome device is added to Home Assistant

2. **State Control**
   - `Tower Status` entity is used to select the signal state

3. **Heartbeat Supervision**
   - `Heartbeat Supervision` switch enables monitoring

4. **Automation**
   - heartbeat automation continuously validates connectivity

---

# Recommended Usage

The provided automation is intended as a **baseline configuration**.

It can be extended to:

- adjust heartbeat intervals
- integrate with specific monitoring systems
- trigger notifications on failure
- implement environment-specific logic

---

# Notes

- The heartbeat is intentionally generated by Home Assistant  
  to ensure true end-to-end connectivity monitoring
- The ESP32 does not generate its own heartbeat  
  to avoid false-positive system states
- Loss of heartbeat always results in a visible fallback signal
